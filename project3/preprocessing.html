Explain the ways to implement preprocessing (client-side vs server-side).

Implementation of preprocessing requires a way to “process” the code into css readable by the browser. This processing can be done in two ways: either on the server itself (before the website is served to the user) or via the client’s browser (as the website is loaded by the user). In the case of using LESS, it can be done server-side by installing LESS on the server hosting the website, or it can be done client-side by including the less.js file with the website and linking it in the HTML. In the first case, the code is converted to CSS before it is loaded in the user’s browser; in the second, it is converted to CSS while it is loaded. Server-side can of course speed up page loading, but it also puts more load on the server.

Define and describe the mechanisms used in either LESS or SASS: mixins, variables, functions, operators, nested variables.

Preprocessors like LESS and SASS come with a variety of features that extend CSS capabilities.

The first is mixins, which simplify CSS coding by including a block of CSS properties in another block. So, if a class is defined to have certain CSS properties, that class can be “mixed in” by adding it to whatever CSS ruleset is intended to use those properties, rather than writing the same code multiple times.

Next, variables also save the user from duplicate code by storing values in one place to be used in multiple locations. Rather than writing the same value over and over (and, if it must be changed, then changing each individual value), the value can be assigned to a variable which is then used in its place. If it is changed, it only needs to be edited from the variable declaration rather than everywhere it is used (because the variable is called in all those places instead).

Functions are provided by LESS to perform specific tasks, often based in parameters input. One example is the if function, which can be used to set a certain value based on a certain condition.

Operators can be used to perform basic mathematical tasks like addition, subtraction, multiplication, or division.

Finally, nesting is useful because it enables CSS blocks to be declared inside other CSS blocks, which is more concise.
